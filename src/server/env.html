mergeloads a js/json file at the path you specify (defaults to '<CWD>/env.js', failing to '<CWD>/env.json'),require()s it, then merges it withprocess.env` favoring values already in place.

Config can be structured like this, either in raw JSON or a commonjs-style .js file (providing values via module.exports):

{
  "NODE_ENV": "development"
  "VAR": "value",
  "VAR2": "value2"
}

All values will be coerced to string as part of being attached to process.env.

Note: we prefer .js files over .json because you can actually put comments in .js files. That's the worst thing about JSON. No comments.
